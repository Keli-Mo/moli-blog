# Gallery 外部图源配置指南

Gallery 页面支持直接从外部图床（如 Cloudflare R2）加载图片，无需下载到本地。

## 工作原理

采用**本地索引 + 按需检测**模式：

1. **`external-index.json`** — 存储已检测到的有效图片 URL 列表
2. **页面加载时** — 只读取 `external-index.json`，**不检测**远端图床
3. **手动更新** — 编辑模式下点击「外部图源」，输入范围检测并更新索引

## 快速开始

### 1. 配置外部图源

1. 进入 Gallery 页面
2. 点击右上角「编辑」按钮
3. 点击「外部图源」按钮
4. 配置 URL 模板（如 `https://cloudflare-imgbed-9ut.pages.dev/file/{n}.webp`）
5. 输入检测范围（如 `1-50`）
6. 点击「开始检测」

### 2. 更新索引流程

点击「开始检测」后：

1. 只检测指定范围内的图片（如 1-50）
2. 存在的图片：**添加**到索引
3. 不存在的图片：**从索引移除**
4. 范围外的图片：**保持不变**
5. 页面**不刷新**，实时更新显示
6. 登录状态下自动推送到 GitHub

### 3. URL 模板

使用 `{n}` 作为数字占位符：

```text
https://your-domain.com/file/{n}.webp
https://your-domain.com/gallery/2025/{n}.jpg
```

### 4. 检测范围说明

- **只检测范围内的图片**：输入 `1-50` 只检测 1.webp 到 50.webp
- **范围外图片保留**：如索引已有 60-70，检测 1-50 不会影响 60-70
- **不存在的自动移除**：范围内检测不存在的图片会从索引中删除

## 工作流程示例

### 初始状态

- `external-index.json` 为空
- Gallery 页面不显示外部图片

### 第一次检测（1-4）

假设只有 1.webp 和 3.webp 存在：

1. 输入范围 `1-4`
2. 检测：1✓, 2✗, 3✓, 4✗
3. 索引更新为：`[1.webp, 3.webp]`
4. 页面显示 2 张外部图片

### 新增图片后检测（5-10）

上传了 6.webp 和 8.webp：

1. 输入范围 `5-10`
2. 检测：5✗, 6✓, 7✗, 8✓, 9✗, 10✗
3. 索引更新为：`[1.webp, 3.webp, 6.webp, 8.webp]`
4. 页面显示 4 张外部图片（1-4范围外的保留）

### 删除图片后检测（1-4）

删除了 3.webp：

1. 输入范围 `1-4`
2. 检测：1✓, 2✗, 3✗, 4✗
3. 索引更新为：`[1.webp, 6.webp, 8.webp]`
4. 页面显示 3 张外部图片（3.webp 被移除）

## 界面说明

### 外部图源按钮

- 显示当前索引中的图片数量
- 示例：`外部图源 (5)` 表示有 5 张外部图片

### 检测对话框

- **当前索引范围**：显示已有的图片编号范围
- **URL 模板**：配置图床地址，使用 `{n}` 占位符
- **检测范围**：输入要检测的起止编号

### 检测进度

- 显示实时检测进度（当前/总数）
- 批量检测，每批 5 个避免阻塞

## 保存方式

### 方式1：登录自动保存（推荐）

1. 点击「编辑」按钮
2. 导入 GitHub App 私钥（.pem 文件）
3. 点击「外部图源」→ 输入范围 → 开始检测
4. 检测完成后自动推送到 GitHub

### 方式2：手动下载（未登录）

1. 执行检测
2. 检测完成后自动下载 `external-index.json`
3. 将文件复制到 `src/app/gallery/external-index.json`
4. 提交并推送到 GitHub

## 注意事项

### 1. 后端检测要求

确保你的图床后端已部署 `/api/exists/{文件名}` 接口：

- 返回 `200`：文件存在
- 返回 `404`：文件不存在
- 支持 CORS 跨域访问

### 2. 检测范围建议

- 范围不宜过大（建议单次检测不超过 200 张）
- 按批次检测，如先 1-50，再 51-100
- 不需要检测整个范围，只检测有变动的部分

### 3. 页面不刷新

检测过程中：

- 页面**不会刷新或闪烁**
- 图片**实时添加/移除**
- 可以同时进行其他操作

### 4. 错误处理

如果图片加载失败（404）：

- 页面会自动隐藏加载失败的图片
- 下次检测该范围时会从索引中移除

## 故障排查

### 检测不到图片

1. **检查 URL 模板**：确保替换 `{n}` 后是完整可访问的 URL
2. **检查后端 API**：确认 `/api/exists/{文件名}` 返回正确的状态码
3. **检查 CORS**：浏览器控制台查看是否有跨域错误

### 不存在的图片仍显示

1. **重新检测**：输入包含该图片的范围重新检测
2. **检查后端**：确认后端 API 正确返回 404 状态码

### 索引未保存

- **未登录**：检测后会自动下载索引文件，需手动上传
- **已登录但失败**：检查控制台错误信息，通常是 GitHub API 问题
