# Gallery 外部图源配置指南

Gallery 页面支持直接从外部图床（如 Cloudflare R2）加载图片，无需下载到本地。

## 工作原理

采用**本地索引模式**：

1. **`external-source.json`** — 存储配置（URL模板、范围、是否启用）
2. **`external-index.json`** — 存储已检测到的有效图片 URL 列表
3. **页面加载时** — 直接读取 `external-index.json`，无需网络检测
4. **手动刷新** — 编辑模式下点击「刷新索引」，检测并更新索引文件

## 快速开始

### 1. 配置外部图源（默认已启用）

1. 进入 Gallery 页面
2. 点击右上角「编辑」按钮
3. 点击「外部图源」按钮（绿色表示已启用）
4. 修改 URL 模板为你的图床地址
5. 点击「保存」
6. 点击「刷新索引」检测可用图片

> **注意**：外部图源默认**已启用**，但首次需要手动刷新生成索引。

### 2. 配置参数

| 参数 | 说明 | 示例 |
|------|------|------|
| **URL 模板** | 图片地址模板，使用 `{n}` 表示数字序号 | `https://cloudflare-imgbed-9ut.pages.dev/file/{n}.webp` |
| **起始数字** | 图片编号的起始值 | `1` |
| **结束数字** | 图片编号的结束值 | `50` |
| **统一描述** | 可选，给所有外部图片添加相同描述 | "旅行照片" |

### 3. URL 模板示例

假设你的 R2 存储桶中图片命名如下：
- `1.webp`
- `2.webp`
- `3.webp`

则 URL 模板为：
```
https://your-domain.com/file/{n}.webp
```

支持多级目录：
```
https://your-domain.com/gallery/2025/{n}.jpg
```

### 刷新索引流程

点击「刷新索引」按钮后：

1. 根据 `external-source.json` 的配置生成 URL 列表
2. 使用 `Image` 对象逐一加载检测（每批5个，避免阻塞）
3. 收集成功加载的 URL
4. 更新页面显示
5. 如果已登录，保存新的索引到 `external-index.json` 并推送到 GitHub

### 混合显示

- **外部图源图片**：显示在瀑布流前面，按索引顺序排列
- **本地上传图片**：显示在外部图源之后
- 左上角提示：`外部图源: X 张 | 本地: Y 张`

### 配置与索引持久化

| 文件 | 内容 | 保存时机 |
|------|------|----------|
| `external-source.json` | URL模板、范围、描述 | 点击「外部图源」→「保存」|
| `external-index.json` | 有效的图片URL列表 | 点击「刷新索引」后自动保存 |

- 登录状态下自动推送到 GitHub 仓库
- 未登录时变更仅在本地生效

## 注意事项

### 1. CORS 跨域

确保你的图床允许跨域访问：
- R2 存储桶设置中开启「公开访问」
- 或在 R2 自定义域上配置 CORS 规则

### 2. 加载数量限制

- 单次最多检测 500 张图片
- 建议范围设置在 1-200 之间，避免检测时间过长

### 3. 图片删除

外部图源的图片无法直接删除，需要：
- 在 R2 存储桶中删除文件，或
- 调整配置范围排除该编号


## 故障排查

### 图片检测不到

1. **检查 URL 模板**：确保 `{n}` 替换后是可访问的完整 URL
2. **检查 CORS**：在浏览器控制台查看是否有跨域错误
3. **检查图片存在**：直接在浏览器访问模板生成的 URL

### 404 图片仍显示

如果检测到 404 图片仍出现在瀑布流，可能是：
- 图床对不存在的图片返回了 200 状态码（带 404 页面）
- 浏览器缓存导致检测不准确

解决方案：
- 手动调整范围跳过不存在的编号
- 等待缓存过期后刷新页面
