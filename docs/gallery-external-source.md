# Gallery 外部图源配置指南

Gallery 页面支持直接从外部图床（如 Cloudflare R2）加载图片，无需下载到本地。

## 快速开始

### 1. 启用外部图源

1. 进入 Gallery 页面
2. 点击右上角「编辑」按钮
3. 点击「外部图源」按钮（紫色/绿色）
4. 开启「启用外部图源」开关

### 2. 配置参数

| 参数 | 说明 | 示例 |
|------|------|------|
| **URL 模板** | 图片地址模板，使用 `{n}` 表示数字序号 | `https://cloudflare-imgbed-9ut.pages.dev/file/{n}.webp` |
| **起始数字** | 图片编号的起始值 | `1` |
| **结束数字** | 图片编号的结束值 | `50` |
| **统一描述** | 可选，给所有外部图片添加相同描述 | "旅行照片" |

### 3. URL 模板示例

假设你的 R2 存储桶中图片命名如下：
- `1.webp`
- `2.webp`
- `3.webp`

则 URL 模板为：
```
https://your-domain.com/file/{n}.webp
```

支持多级目录：
```
https://your-domain.com/gallery/2025/{n}.jpg
```

## 工作原理

### 自动检测机制

启用后，系统会：
1. 根据配置的 URL 模板和范围生成所有可能的 URL
2. 使用 `fetch HEAD` 请求逐一检测图片是否存在
3. 只加载实际存在的图片到瀑布流
4. 左上角显示检测进度和结果统计

### 混合显示

- **外部图源图片**：显示在瀑布流前面，按编号顺序排列
- **本地上传图片**：显示在外部图源之后
- 左上角提示：`外部图源: X 张 | 本地: Y 张`

### 配置持久化

- 配置保存在 `src/app/gallery/external-source.json`
- 登录状态下点击「保存」，配置会同步到 GitHub 仓库
- 未登录时配置仅在当前浏览器生效

## 注意事项

### 1. CORS 跨域

确保你的图床允许跨域访问：
- R2 存储桶设置中开启「公开访问」
- 或在 R2 自定义域上配置 CORS 规则

### 2. 加载数量限制

- 单次最多检测 500 张图片
- 建议范围设置在 1-200 之间，避免检测时间过长

### 3. 图片删除

外部图源的图片无法直接删除，需要：
- 在 R2 存储桶中删除文件，或
- 调整配置范围排除该编号

### 4. 缓存机制

- 每次刷新页面都会重新检测
- 未来版本将添加本地缓存避免重复检测

## 故障排查

### 图片检测不到

1. **检查 URL 模板**：确保 `{n}` 替换后是可访问的完整 URL
2. **检查 CORS**：在浏览器控制台查看是否有跨域错误
3. **检查图片存在**：直接在浏览器访问模板生成的 URL

### 404 图片仍显示

如果检测到 404 图片仍出现在瀑布流，可能是：
- 图床对不存在的图片返回了 200 状态码（带 404 页面）
- 浏览器缓存导致检测不准确

解决方案：
- 手动调整范围跳过不存在的编号
- 等待缓存过期后刷新页面
